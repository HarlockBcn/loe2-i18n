## 1. Text Extraction

The script **dllscript/dll_dir_sub_extractor.py** was created, capable of extracting texts from all **dll** files present in the folder passed as a parameter.

```
.\src\dllscript\dll_dir_sub_extractor.py ${lone-echo2-game-path}\bin\win10\scripts output\scripts_dll_subs_english.txt
```

The generated file format is as follows:

```
{filename}|{va_address}|{file_offset}|{max_size}|{text}
```

Where:
- **filename**: DLL file name (without extension)
- **va_address**: va_address within the DLL where the text literal was found
- **file_offset**: offset within the DLL file where the text literal is located
- **max_length**: maximum allowed length for the text literal. (Editing a DLL, we are limited). Note that the length is in bytes and some characters encoded in **utf-8** take 2 bytes.
- **text**: text

Example content:
```
0001cf5152080030|0x180007300|0x6700|39|Liv. That's not going to happen to you.
0001cf5152080030|0x18000706c|0x646c|7|Yeah.
0001cf5152080030|0x1800072d0|0x66d0|47|Whatever it takes, we'll find a solution.
0001cf5152080030|0x1800072c0|0x66c0|15|I know Jack.
001562b639bcb14f|0x1800082b8|0x6ab8|39|Lucky for us, it was still functional.
```

## 2. Manual Filtering

Once the file with all texts is obtained, a manual process is performed to remove non-translatable texts: those containing variables or that could cause problems.
(Unfortunately, there is no automation for this process)

## 3. Text Translation

At this point, you can start translating all the subtitle text files from the English version.

**IMPORTANT**: 
    - Only modify the text literal part, never modify the numbers and identifiers.
    - Use **UTF-8** encoding

### AI Automation
For the Spanish translation, batch processing using AI was used, reviewing the result iteratively (it can always be improved).

In this case, a single step was used with the prompt [translate-redux-dll-subs-spanish.md](../../prompts/translate-redux-dll-subs-spanish.md), which requests a translation under rules for the maximum length of the texts.

### QA and Helper Scripts

A series of scripts were created in `src/dllscript` to assist in subtitle processing.
The quality of the scripts is poor (they were made quickly and with AI), but they are fully functional and were very useful.

Start from a subtitle file where each line has this format:

```
{file_id}|{va_address}|{file_offset}|{max_length}|{length}|{text}
```

This is the format generated by our AI prompt used for translation. Where:

- `{filename}`: DLL file name (without extension) of the text.
- `{va_address}`: address within the DLL of the text.
- `{file_offset}`: offset in the DLL file where the text goes.
- `{max_length}`: length limitation in bytes.
- `{length}`: number of bytes occupied by `{text}` encoded in **utf-8**.
- `{text}`: translated text.

**General Help**
- **merge_subs_by_key**: allows merging two subtitle files.

**QA**
- **QA/fix_sub_lengths.py**: recalculates the `{translated_length}` field and reports cases where the translation length exceeds the `{max_length}` limit.
- **QA/fix_sub_lengths_utf8.py**: checks that all subtitle identifiers in the translated file are in the original, to find altered ids that may have been made by the AI, for example.

## 4. DLL Script Patching

Start from a file with the following format:
```
{filename}|{va_address}|{file_offset}|{max_length}|{length}|{text}
```

The script `dllscript/dll_dir_sub_patcher.py` was created:
```
python ./src/dllscript/dll_dir_sub_patcher.py
usage: dll_sub_patcher.py <txt_file> <dlls_folder>
```

This script patches the DLL files in the indicated folder, replacing the original texts with the translated ones.

If you run:
```
src/dllscript\dll_dir_sub_patcher.py  translated_subs_file.txt ${lone-echo2-game-path}\bin\win10\scripts
```

you will patch the DLL files of the game containing these texts.

(It is recommended to back up the original `scripts` folder of the game)
